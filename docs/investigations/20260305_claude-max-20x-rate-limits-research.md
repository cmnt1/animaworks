# Claude Max 20x プラン レート制限調査レポート

**調査日**: 2026年3月5日  
**目的**: Claude Max 20x プランの Opus と Sonnet のトークン配分・レート制限の違い、5時間/7日ウィンドウの適用、2025-2026年の変更、コミュニティ体験を把握する。

## 1. モデル別トークン消費比率

### Opus vs Sonnet の配分差

| 項目 | 内容 |
|------|------|
| **消費比率** | **Opus は Sonnet の約 5倍** の割当を消費する |
| **根拠** | API価格差（Opus: $5/$25 per M tokens、Sonnet: $3/$15 per M tokens）に比例 |
| **出典** | ClaudeLog、Awesome Agents、Medium (Red Asgard) |

Max プランはトークン課金ではなく定額だが、内部の割当計算では API 価格に相当する重み付けが使われているとされる。

### Haiku / Sonnet / Opus の相対消費

- **Haiku**: 最も軽量
- **Sonnet**: 中間（Haiku の約 3倍）
- **Opus**: Sonnet の約 5倍、Haiku の約 3–5倍

同一タスクを Opus と Haiku で実行した実験では、Opus が約 4倍の割当を消費したという報告あり（Medium）。

## 2. 5時間ウィンドウと7日ウィンドウ

### 二重制限システム

Claude Max は **5時間ローリングウィンドウ** と **7日間週次キャップ** の2種類の制限を持つ。

| 制限種別 | 仕組み | リセット |
|----------|--------|----------|
| **5時間ウィンドウ** | 最初の利用から5時間のローリングウィンドウ | 最初のトークン使用から5時間後に古い分が落ちる |
| **7日間キャップ** | 週次ハードキャップ | 7日ごとにリセット |

### 5時間 vs 7日の違い

- **5時間制限に達した場合**: 5時間待てば再開可能
- **7日間制限に達した場合**: 7日間リセットまで待つ必要がある（5時間が空いていても使えない）
- **共有プール**: Web（claude.ai）、Claude Code、Chrome 拡張が **同一の割当を共有**

### 5時間ウィンドウの注意点

- 固定時刻ではなく「最初のトークン使用から5時間」で計算
- 例: 14:00 に開始 → 19:00 に古い分が落ちる（0:00 ではない）
- バースト使用後は、最後のトークンからではなく **最初のトークンから5時間** 経過するまでブロックされる

## 3. プラン別の具体的な数値

### 5時間セッションあたりのメッセージ数（目安）

| プラン | 価格 | 5時間あたりメッセージ数（短いメッセージ・軽いモデル時） |
|--------|------|--------------------------------------------------------|
| Max 5x | $100/月 | 約 225 メッセージ |
| Max 20x | $200/月 | 約 900 メッセージ |

※実際の数はメッセージ長・ファイルサイズ・会話長・プラットフォーム負荷で変動。

### 週次時間制限（Claude Code 向け）

| プラン | Sonnet 4 週次時間 | Opus 4 週次時間 |
|--------|------------------|-----------------|
| Pro | 40–80 時間 | （Web のみ、Code では Sonnet のみ） |
| Max 5x | 140–280 時間 | 15–35 時間 |
| Max 20x | 240–480 時間 | 24–40 時間 |

※「時間」は実時間ではなく、使用パターンに基づく換算値。実際の消費はトークン量で決まる。

### 7日と5時間の適用

- **5時間ウィンドウ**: 全モデル共通（Haiku / Sonnet / Opus とも同じルール）
- **7日間キャップ**: 全モデル・全プラットフォームで **1つの週次プール** を共有
- モデルごとの別枠ではなく、**Sonnet と Opus が同じ週次プールから消費** する（Opus の方が重いため、実質的に Opus 時間が早く尽きる）

## 5. コミュニティの体験

### Opus 4.6 の消費増加

- **Max 5x ($100/月)**: 15–45分でセッション制限に達したとの報告が複数
- **Max 20x ($200/月)**: 2時間未満で制限に達したとの報告
- **具体例**:
  - 通常 10% 程度の消費だった研究タスクが、Opus 4.6 で **90%** を消費（約9倍）
  - 30–40ファイルの diff 分析＋コミットメッセージ生成で、**1リクエストで5時間枠の 48%** を消費（以前は約15%）
  - Max 20x ユーザーが12時間以内に **週次使用量の 20%** に達したとの報告

### Opus 4.6 が重い主な理由

1. **Adaptive thinking（常時オン）**: 内部推論トークンが出力トークンとして課金され、1リクエストあたり 2–5倍のトークン生成
2. **100万トークンコンテキスト**: 200K 超でプレミアム価格（$10/$37.50 per M）相当の重み付け
3. **バックグラウンドエージェント**: Claude Code v2.1.0 以降、3体中2体が Opus を使用（会話がなくても消費）

### コミュニティの対策例

- Opus 4.5 に固定して Opus 4.6 の自動アップグレードを避ける
- `/effort` で thinking を medium / low に下げる
- `MAX_THINKING_TOKENS=8000` で thinking トークンに上限を設ける
- `/clear` でタスク間のコンテキストをリセット
- タスクに応じて Haiku / Sonnet / Opus を切り替える（Haiku: 軽い作業、Sonnet: 実装、Opus: 設計・複雑な分析）

## 7. まとめ

| 項目 | 結論 |
|------|------|
| **Opus vs Sonnet の配分** | Opus は Sonnet の **約5倍** の割当を消費 |
| **5時間 vs 7日** | 5時間は全モデル共通のローリングウィンドウ。7日は全モデル・全プラットフォームで共有の週次キャップ |
| **モデル別の別枠** | 週次キャップは Sonnet/Opus で別枠ではなく、**同一プール** から消費（Opus の方が重い） |
| **2025–2026の傾向** | 7月の静かな引き締め、8月の週次キャップ導入、9月の大幅厳格化、Opus 4.6 による 3–9倍消費増加 |
| **コミュニティの不満** | 制限の不透明さ、事前告知のない変更、Opus 4.6 の消費増加に対する不満が多数 |
